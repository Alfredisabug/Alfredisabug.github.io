問的方式 X-Y Problem

Publish Date : 2018-09-28 09:42

自從進公司後
一直在增加技能樹的空間
透過一直挖掘
才能成長阿~

軟體工程的世界
真的是博大精深
越挖越發現
學校教的其實根本不夠用
更多的是根本沒教到

因此要成為一個工程師
要會記得google阿~
ㄚㄚ還有英文!
但更多時候會用到stack overflow
或是一些論壇上發問
這時候就會衍伸一堆問題

X-Y Problem便是其中一種

其實 X-Y Problem定義很簡單:
1）有人想解決問題X
2）他覺得Y可能是解決X問題的方法
3）但是他不知道Y應該怎麼做
4）於是他去問別人Y應該怎麼做?

最後的結果可能會導向，別人告訴你Y怎麼做了
結果還是不能解決X，所以你再講了X的問題
別人才發現Y其實不能解決X，不應該使用Y!

X-Y Problem最大的嚴重的問題就是：
在一個根本錯誤的方向上浪費他人大量的時間和精力！

X-Y Problem其實是可以避免的
發問時，不要只提Y，可以嘗試著提起X
但不要直接問說X要怎麼處理
這樣應該會被噴慘

發問時
想好要怎麼問
才不會浪費了雙方彼此的時間
發問的藝術阿

另補一個金句:
Intellectual yet idiot. (IYI)
